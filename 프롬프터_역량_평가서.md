```

```

- **평가**: 참조 기준(전체화면 비율)을 명시하여 의도 전달
- **점수 기여**: +10점

- **점수 기여**: +8점

### 1.2 개선 필요 사례 (실제 대화 인용)

#### ⚠️ 점진적 요구사항 변경
```
1회차: "도서 상세 페이지에서 별이 겹쳐서 나오는거 수정해줘"
2회차: "2분의 1표시는 없었으면 좋겠어 별만 있게"
3회차: "반별 표시를 할거면, 반별 이모지를 사용했으면 좋겠어"
4회차: "그냥 반별 기능 없애줘"
5회차: "반별 관련된거 지워줘 안쓸거니까 필요한거야?"
```
- **분석**: 
  - 1회차: 겹침 문제만 수정 → 반별 유지 의도로 해석
  - 2회차: ½ 문자만 제거 요청
  - 3회차: 반별 이모지로 변경 (⯪)
  - 4회차: 반별 기능 전체 제거
  - 5회차: 평균 평점의 반별까지 제거 확인
- **영향**: 5회 수정이 1회로 가능했음
- **점수 감점**: -12점

#### ⚠️ 모호한 응답
```
AI: "GitHub에도 푸시할까요?"
사용자: "아 엉"
```
- **분석**: "응"인지 "아니"인지 해석 필요 (문맥상 "응"으로 처리)
- **점수 감점**: -3점

#### ⚠️ 실시간 의사결정 변경
```
사용자: "그리고 답글 그 책 고른 사람 컬러로 해달랬던거 해줘 적용 안된듯"
(수정 진행)
사용자: "아 아니다 없는게 나을듯 답글 색이나 배경 관련 코드 없애줘"
```
- **분석**: 요청 → 수정 완료 → 취소의 패턴
- **점수 감점**: -5점
